NP=P？

2025-10-16 2025-10-16

要了解NP问题，首先得了解时间复杂度。

在计算机科学中，算法的时间复杂度（time complexity）是一个函数，它定性描述该算法的运行时间。这是一个代表算法输入值的字符串的长度的函数。时间复杂度常用大O符号表述，不包括这个函数的低阶项和首项系数。使用这种方式时，时间复杂度可被称为是渐近的，亦即考察输入值大小趋近无穷时的情况。

时间复杂度并不是表示一个程序解决问题需要花多少时间，而是当问题规模扩大后，程序需要的时间长度增长得有多快。也就是说，对于高速处理数据的计算机来说，处理某一个特定数据的效率不能衡量一个程序的好坏，而应该看当这个数据的规模变大到数百倍后，程序运行时间是否还是一样，或者也跟着慢了数百倍，或者变慢了数万倍。(来自博客)。

时间复杂度是一个定性的函数，而不是考虑一个算法解决问题的具体时间，而是根据时间增长的快慢来判断一个算法的优劣。

常见的时间复杂度

  • 多项式级：O(nk),O(log(n))O(n^k),O(log(n))O(nk),O(log(n))
  • 非多项式级：O(an),O(n!)O(a^n),O(n!)O(an),O(n!)

时间差不多喽，开始吧

P问题（polynomial time class）：是指在多项式时间内，可以找出解的决定性问题(decision problem)。这个很好理解，能找到多项式级时间复杂度解法的问题就是P问题。

那么什么是NP问题（non-deterministic polynomial）呢？先看英文，deterministic的意思是确定的，所以整个短语的意思是不能确定是否可以在多项式时间内找出解。

举个例子，售货员旅行问题 （traveling salesman problem）就是经典的NP问题。 有一个售货员要开汽车到 n 个指定的城市去推销货物，他必须经过全部的 n 个城。 现在他有一个有此 n 城的地图及各城之间的公路距离，试问他应如何取最短的行程从家中出发再到家中？

很显然，最直接的方法是遍历所有路径，那么时间复杂度就来到了惊人的O(n!)O(n!)O(n!)，有没有更快的算法呢？有的，目前最优的精确算法（Held–Karp 动态规划）时间复杂度是 O(n22n)O(n^2 2^n)O(n22n)，比 n!n!n! 小得多，但仍是指数级。

那么有没有更简单的算法呢？能让时间复杂度来到多项式级。很遗憾的是，至今尚未发现。如果聪明的你发现了，那么恭喜你，解开了困扰了世界半个世纪的千禧年难题NP=P？并且可以获得100万美元的奖金。

好的，虽然你狠激动，但是让我们等下再来说这个难题，现在先暂且回到旅行家的身边。虽然我们找不到更简单的算法，但是我们有一个更简单的方法，猜！现在的你欧皇附体，鸿运齐天蛊加身，随便指了一条路竟然恰好就是答案，太amazing了，假设我们事先已经知道答案，那么我们验证答案就只需要O(n)O(n)O(n)的时间复杂度了。所以虽然我们没法在多项式时间了找出解，但是我们可以在多项式时间里验证一个解。

恭喜你，你已经了解了NP问题的一个定义了。维基百科：NP问题包含可在多项式时间内验证其解是否正确，但不保证能在多项式时间内能找出解的决定性问题。

你能很轻松地发现，所有P问题都是NP问题，如果你能在多项式的时间里找到解，那么在多项式时间里验证一个解是轻而易举的。但是，所有NP问题都是P问题吗？

先让我们介绍规约的概念，规约就是将某个计算问题变换为另一个问题的过程，注意，这个过程是在多项式时间下进行的变换。举个例子，如果一个问题B的解法可以用来求解问题A，那么称为A可以规约到B。直观的理解一下，一个问题不能找到更简答的解法，但是或许能找到一个更复杂的解法来包含它，那么就可以规约到一个更加复杂的问题。

而且规约有传递性。那么这么来说，随着问题的不断规约，复杂度不断加大，应用范围也在不断增大。自然地，如果不断地规约上去，不断找到能通吃小NP问题的大NP，最后是否能找到一个终结的NP问题？答案竟然是肯定的。存在这样一个NP问题，所有NP问题都能规约成它。而且如果解决这个问题，所有NP问题就都解决了。那么如果这个问题是P问题，那么NP=P就确定了。更不可思议的是，这种问题不止一个，而是一类，它们被称为NPC问题（NP-Complete）。它是NP中最难的问题。

那么NPC问题的具体定义是什么呢？先让我们介绍一个基本的问题，SAT（满足问题）

令 x 1，x 2，…，x n 代表布尔变量(boolean variables)(其值非真(true)即假(false)的变量)。令-xi 代表 xi 的相反数(negation)。一个布尔公式是将一些布尔变量及其相反数利用而且(and)和或(or)所组成的表达式。满足问题是判断是否存在一种指定每个布尔变量真假值的方式，使得一个布尔公式为真。

再介绍一个概念，NP-hard问题：如果一个问题可以由满足问题转化而来，那么这个问题称为是NP-hard的。

那么NPC呢？一个既是NP问题，又是NP-hard的问题就是NPC问题。

因为NPC的存在，让人们更加难以相信信NP=PNP= PNP=P，因为把NPC问题转化成P问题太不可思议了。

最后再来说一下NP-hard问题：它和NP问题是两个集合，交集称为NPC。

其中一个经典的问题，围棋问题：以平常的围棋规则在一个 n x n 的棋盘上下，给定一个残局（下了二个子就可以算残局），首先，是否可以确定黑子在最好的下法之下，一定会赢？
